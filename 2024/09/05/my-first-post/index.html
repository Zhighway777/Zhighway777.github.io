How to build personal website

By Zeng

September 5, 2024

Introduction

第一篇博客用于记录博客的搭建过程

模板

博客使用hugo工具进行搭建与管理，hugo提供了丰富的tamplates,同时，其draft功能使得博客网站的开发更加简单。开发者这需要关注博客的内容本身，而不必特地学习CSS或HTML进行网站搭建，除此之外，博客内容则使用Markdown格式记录，方便不同平台的移植。

hugo 使用方法

具体参阅Hugo-quick-start

博客内容在/content/posts文件夹下，以.md的格式存放，每增加一个.md文件，则在博客中增加一个子页面。 在.md开头部分有博客的基本信息：题目，日期，和draft状态。 drift默认被设置为ture,说明此时是草稿状态。

保存draft后，在命令行使用 hugo server -D可以打开draft,进行实时修改。

当修改后的内容符合要求，可以将draft调整为false，即可保存修改并对修改后的内容进行publish。 最后使用hugo命令发布网站。这会在本地publsh中更新相应的内容，但不会上传到真正的网站上。

由于我通过将hugo接入github.io 因此需要再将内容使用git推送后，可以将更新内容发布。

关于博客内容

I will try my best to use English convert something clearly 我尽量使用英文来进行文章书写，并且保证语句的简单和格式的清晰。

只输出有价值/经过思考/有趣的观点或见解。

尽量不包含主观情绪，不涉及任何政治话题。

使用奥卡姆剃刀原理来部署博客内容

Entities should not be multiplied unnecessarily (如非必要，勿增实体)
