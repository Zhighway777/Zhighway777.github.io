yield

  1. 如果一个函数中使用yield来修饰一变量，那么这个函数会作为一个generate(), 当调用此函数的时候，并不会运行此函数，而是返回一个generate对象
  2. 对于这个函数(对象)，使用next()方法让其开始运行。
  3. 遇到yield就相当于遇到return, 这时将返回一个结果，但区别是函数会记录执行到此位置，下次接着从yield所在的地方开始执行()
  4. 再次调用next(), 从上一次yield的地方开始运行，知道执行到下一次yield的地方。
  5. 可以使用for语句来[循环迭代 生成器 (generate class)]， 逐个获取generate()的返回值。同时，for循环内部自动处理停止迭代异常。
  6. 当生成器中的数据全部生成完毕(后续再也没有yield指令或者循环结束)，再次调用next(gen)的时候，会抛出StopIteration异常，表示数据生成完毕。
def func():
	for i in range(10):
		yield 1 * 1
gen = func()


#1
print(func)
#输出<generator object generator at 0x0000....>


#2,3
print("frist time")
print(next(gen))
#输出0


#4
print("second time")
print(next(gen))
#输出1


#5
for value in func():
	print(value)
#输出 4, 9, 16, .... , 81

#6
print(next(gen)) 
#抛出 StopIteration 异常 Process finished.

yield from

yield from 语法可以简化生成器函数的定义。yield from 可以将一个嵌套的生成器的值逐个返回给调用者，而不需要手动遍历嵌套生成器的每个值。例如：

def my_generator():
    yield from [1, 2, 3]
    yield from (4, 5, 6)
    yield from my_other_generator()

def my_other_generator():
    yield "a"
    yield "b"
    yield "c"

在这个例子中，my_generator 函数使用 yield from 将一个列表、一个元组和另一个生成器的值逐个返回给调用者。调用 my_generator 函数时，它会返回一个生成器对象，该对象将逐个返回下列值：1、2、3、4、5、6、“a”、“b” 和 “c”。

getattr()

这是一个python的内建函数，python存在很多内建函数(Built-in function).这里罗列了大部分的内建函数，可供查阅。 使用方式： getattr(object, name, default[optional]) 用于提取某个对象的属性，注意name必须是以字符串的形式出现。如果name所指示的属性不存在，那么以default作为返回值。当没有设置default时，而且name所指示的属性不存在，那么抛出AttributeError错误。

getattr(x, 'foobar') #<==> x.foobar

enumerate()

(对C语言中的 enum有印象吗？)enumerate(iterable, start=0)将一个可以迭代或者可以作为一个序列的对象变为具备序号的元组。看下边的例子：

>>> seasons = ['Spring', 'Summer', 'Fall', 'Winter']
>>> list[enumerate(seasons)]
[(0, 'Spring'), (1, 'Summer'), (2, 'Fall'), (3, 'Winter')]
>>> list[enumerate(seasons, start=100)]
[(100, 'Spring'), (101, 'Summer'), (102, 'Fall'), (103, 'Winter')]

更进一步地，其实enumerate()等效为：

def enumerate(iterable, start=0):
	n = start
	for elem in iterable:
		yield n, elem
		n += 1

get()

字典中，访问一个key对应的value， 如果key存在，那么直接访问对用value， 如果key不存在，那么访问get()设置的默认值。【不会改变dict】

get(key, defaultValue)
spam.get('eggs', 0)

defaultValue: 可以保证在进行dict访问时不出现错误，同时输出默认value

setdefault()

为某一个key设置默认值， 当这个key没有对应的value时，使用这个默认值，并将其添加到dict中。【会改变dict】

spam = {'name' : pika, 'age' : '42'}
spam.setdefault('color', 'black')
>>>spam
{'color': 'black''name' : pika, 'age' : '42'}

#对已经存在的key设置默认value，不会改变dict
spam.setdefault('color', 'whithe')
>>>spam
{'color': 'black''name' : pika, 'age' : '42'}

field

允许为数据类的每个字段设置额外的选项，比如默认值，字段比较，初始化是是否包含构造函数等。


@dataclass
class SubClass


@dataclass
class MyClass
	x:int
	y:List[int] = field(default_factory=list) 

obj1=MyClass(x=1)
obj2=MyClass(x=2)

obj1.y.append(100)
print(obj1.y)
print(obj2.y)

这里default_factory=list 确保了 y 字段在每次实例化时是一个新的、独立的列表，而不是多个实例共享同一个对象。
