Although is funny, it actually describes the OOP concept.

类与对象

这是一个is-a关系，an object is - a class 也就是说 一个 对象(object) 可以拥有一个类(class)的各种属性和方法， 同时一个类可以包含多个对象， 因此可以将类看作对象(object)的属(var)性和行为(func)的抽象表示。这里一个具体的对象也成为类的一个实例。

方法， 数据成员

这是一个类中所表象出来的具体特征，数据成员表示在一个类中所包含的属性，方法表示一个类可能的行为，这可能会对类中的属性进行组织和管理。

  1. 方法根据访问的类别（实例， 类）可以分为：

    实例方法（普通方法）：默认有self参数，访问具体对象的属性(实例变量)，一般作用于一个实例上。

    类方法：默认有cls参数，可以访问实例或者类的属性(类变量和是实例变量)，需要加上 @classmethod 装饰器，一般作用于类本身。

    静态方法：没有self或者cls默认参数，不依赖于实例或者类，只执行某种逻辑， 使用@staticmethod 装饰。

  2. 数据成员是【类】或者【将类实例化】的【对象】中的属性，根据其可以被访问的范围不同，可以分为：

    类变量：类变量是在class定义的属性，可以被所有实例共享。

    实例变量：属于一个具体的对象，被这个实例对象独占。

类的定义，实例化以及对象的引用

 #类的定义
class Dog:
	species: "Canine" #类变量（也称属性）
	def bark(self): #实例方法
		return ("I am a Dog, woof!")

类的构造方法

在对类进行实例化的时候，可以使用向类中传入参数，通过类的构造函数来对实例化后的对象进行初始化，方便构造对象的属性。这个构造函数使用__init(self, arg0, …)__在类中进行定义。

class Dog:
	species: "Canine" #类变量
	
	#构造函数，当实例化时需要同时传入self后的参数
	def __init__(self, n, a):
		self.name = n #实例变量
		self.age= a
	
	def bark(self): #实例方法
		return (f"I am a Dog, woof!")
		
	def introduce(self)
	 return (f"My name is {self.name}, and I am {self.age} years-old")
	
dog_1 = Dog("Max", 4) #传入构造函数的参数
dog_2 = Dog("Luke", 5)

dog_1.introduce()
dog_2.introduce()

'''注释
f 表示 f-string（格式化字符串），它是一种将变量值直接嵌入字符串中的简洁方式。
f-string 通过在字符串前加上字母 f，然后在字符串中使用大括号 {} 来包含变量或表达式，
从而将变量的值动态插入到字符串中。
'''

在python3.7++版本，使用@dataclass可以快速定义__init__()构造函数

类的私有属性和私有方法

在2.类的方法和数据成员中，其实可以定义私有的属性和方法这里，一般由两个下划线”__”来表征。

 #类的定义
class Dog:
	species: "Canine" #类变量（也称属性）
	__host: "Jasper" #私有属性，只能在类中被访问，在外部无法被访问
		
	#私有方法
	def __private_bark(self)
		print("a private woof!")
		
	def foo(self) #实例方法，公共方法
		#在公共方法中调用私有方法, 这里需要加【self.】表示时在本class内部的方法
		self.__private_bark() 
		#在类中访问类的私有属性
		print(f"Its host is {self.__host}")
	
	def change_host(self)
		#在类中访问并【改变】类的私有属性
		self.__host = "Marry"
		print(f"Its host is {self.__host}")
		
 #类的实例化
dog_1 = Dog()
通过公共方法访问私有方法和私有属性
dog_1.foo()
dog_1.change_host()

#以下会报错，不存在该属性
print(dog_1.__host)
dog_1.__private_bark()

由此可以看出，其实构造函数__init__()也是私有方法，这其实时【类的专有方法】的一种。专有方法有许多，可以参考：https://blog.csdn.net/amber_o0k/article/details/90737799

比较常见的有：

__init__ : 构造函数，在生成对象时调用
__del__ : 析构函数，释放对象时使用
__len__: 获得长度
__str__:输出类中默认的字符串

类的继承

通过类的继承，可以快速派生出多个不同的类，进而灵活地构造各种不同的对象。继承时类的一个非常重要的特性。在类的继承中，根据主体不同，分为派生类和基类，如果一个类A继承了另一个类B的属性和方法，那么称A为派生类，B为基类。

子类（Child class\派生类 DerivedClassName）

父类（Parents class\基类 BaseClassName）

#BaseClass
class Dog:
	species: "Canine" #类变量
	__host: "" #私有变量
	#构造函数
	def __init__(self, n, a, h):
		self.name = n #实例变量
		self.age = a
		self.__host = h
		
	def introduce(self) #实例方法
	 return (f"My name is {self.name}, and I am {self.age} years-old")

#DerivedClass
class BeagleDog(Dog):
	weight = 0
	#建立子类的构造函数
	def __init__(self, n, a, h, w):
		#调用父类的构造函数
		Dog.__init__(self, n, a, h)
		#构造子类独有的属性
		self.weight = w
	
	#注意这里的函数和父类中的函数同名，因此会重写overwrite该方法
	def introduce(self)：
		print(f"My name is {self.name}, and I am {self.age} years-old.")
		print(f"My host is {self.__host}, and my weight is {self.weight} kg.")
		
dog_1 = BeagleDog("Max", 1, "Jasper", 3)
dog_1.introduce()
#输出如下：
# My name is Max, and I am 1 years-old.
# My host is Jasper, and my weight is 3 kg.

关于继承的其他说明：

  1. 在 Python 中定义派生类（子类）时，派生类的基类（父类）不必与派生类在同一个文件（模块）中定义。可以在不同的模块中定义基类，然后通过导入模块来继承该基类。

    这对于模块化编程非常有用，因为你可以将不同类的定义分散到不同的模块（文件）中，保持代码的组织性和可维护性。通过 import 语句，你可以从其他模块导入基类，并在当前模块中定义派生类

  2. 除了直接从模块导入类，还可以用表达式定义基类，这意味着你可以动态决定哪个类作为基类。这在某些高级场景中可以实现灵活的继承模式。

 假设我们有两个基类模块
class Base1:
    pass
class Base2:
    pass
    
# 使用表达式选择哪个类作为基类
def get_base(use_base1):
    return Base1 if use_base1 else Base2

# 动态选择基类
class Derived(get_base(True)):  # 这里会继承 Base1
    pass

类的多继承

#BaseClass 0
class Dog:
	species: "Canine" #类变量
	__host: "" #私有变量
	#构造函数
	def __init__(self, n, a, h):
		self.name = n #实例变量
		self.age = a
		self.__host = h
		
	def introduce(self) #实例方法
	 return (f"My name is {self.name}, and I am {self.age} years-old")

#BaseClass 1 另一个基类
class Animal:
	feets = 0
	
	def __init__(self, n, f, t):
		self.name = n #与BaseClass 0的属性冲突
		self.feets = f
		self.has_tail = t
		
	#与BaseClass 0的方法名称冲突
	def introduce(self):
		print(f"My name is {self.name}. I have {self.feets} feets, and my tail is {self.tail}. ")
		
#DerivedClass
class BeagleDog(Animal, Dog):
	weight = 0
	age = 0
	
	#建立子类的构造函数
  def __init__(self,n,f,t,a,h,w):
     Animal.__init__(self,n,f,t)
     Dog.__init__(self,n,a,h)
     self.weight = w
     self,age = a
     
dog_1 = BeagleDog("Max", 4, True, 1, "Jasper", 4)

#方法名同，默认调用的是在多继承子类的括号中参数位置排前父类的方法
#class BeagleDog(Animal, Dog)  也即是Animal的方法

dog_1.introduce()
#输出如下：
# My name is Max. I have 4 feets, and my tail is Ture.

方法重写

在派生类中，如果基类的方法不满足要求，可以进行重写，接着上面的代码：

#DerivedClass
class BeagleDog(Animal, Dog):
	weight = 0
	age = 0
	
	#建立子类的构造函数
  def __init__(self,n,f,t,a,h,w):
     Animal.__init__(self,n,f,t)
     Dog.__init__(self,n,a,h)
     self.weight = w
     self,age = a
     
	#方法重写
	def introduce(self)
		print("woolf, woolf, woolf!")
		
dog_1 = BeagleDog("Max", 4, True, 1, "Jasper", 4)
dog_1.introduce()
#输出如下：
# "woolf, woolf, woolf!

以上关于子类继承父类构造函数的情况有些复杂，这里稍微总结一下：

情况一：子类需要自动调用父类的方法：子类不重写__init__()方法，实例化子类后，会自动调用父类的__init__()的方法。

情况二：子类不需要自动调用父类的方法：子类重写__init__()方法，实例化子类后，将不会自动调用父类的__init__()的方法。

情况三：子类重写__init__()方法又需要调用父类的方法：使用super关键词：

super(子类，self).__init__(参数1，参数2，....)
class Son(Father):
  def __init__(self, name):
    super(Son, self).__init__(name)
