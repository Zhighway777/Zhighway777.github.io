outstanding 可以理解为：发出但未被响应

  1. This concept comes from Networks

  2. https://blog.csdn.net/hit_shaoqi/article/details/53243173

    而如果outstanding能力为N>1的话，则： 读操作：可以连续发N个读地址命令，这期间如果读数据没有返回，则需要需要等待读数据返回，如果有读数据返回，则返回了几个，那么仍然可以接着发几个。也就是说，“在路上” 的读命令（或者读数据）最多可以是N。多说一点，可以看出，如果数据返回得比较慢，那么IP需要等待，效率就会比较低，因此，为了提高效率，有必要提高outstanding能力，以弥补”路上“（总线）引入的延时。但是也不能无限制地发，有可能会引起总线拥塞，把其他IP给堵住。 写操作：可以连续发出N组写地址（写数据）命令，这期间如果写响应没有返回，则必须等待写响应返回才能接着发写地址（写数据）命令，如果有写响应返回，则返回了几个，那么仍然可以接着发几组。也就是说，“在路上” 的写响应最多可以是N。

  1. NVLink

核心优势在于它能够绕过传统的 CPU 分配和调度机制，允许 GPU 之间进行直接的数据交换。

NVLink 和 PCIe 可以作为互补的解决方案共存

NVLink 不仅仅是一种点对点的通信协议，它还可以通过连接到 GPU 内部的交换机（XBARs）来实现更复杂的连接拓扑。

  1. NVSwitch

    https://chenzomi12.github.io/02Hardware04NVIDIA/06DeepNvswitch.html

    允许多个 GPU 之间实现全互联

    值得注意的是， 在NVlink和NVswitch的演进过程中， 考虑的是GPU之间的通信， 进一步细化为：XBar， SM， HBM， PCIe， 以及CPU之间的关联。

    特点：

    1. 多个GPU互联能力提升
    2. 单一的GPU可控制全局GPU集群
    3. 无干扰的HBM访问
    4. XBAR的带宽提升 （桥接器）

  2. PCIe

    PCIe 是一种高速串行计算机扩展总线标准，用于连接主板上的硬件设备。但是，由于 PCIe 的带宽有限

  3. CXL

    Compute Express Link 相当于PCIe 6.0的拓展

    1. CXL.io: 改进的PCIe协议 ，用于初始化， 链接和设备发现以及寄存器访问。为I/O设备提供了非一致的加载/存储接口。

    2. CXL.cache: 允许设备高效的对主机进行缓存， 加载和存储命令访问设备连接的内存

    3. CXL.mem: 提供了主机处理器， 支持易失性和持久性存储器架构。

  1. shmoo: 图形化方法，用于展示电路或芯片的性能特性 （如电压、频率等参数）与其成功或失败状态之间的关系。

    偏置电压和时钟频率是可变的，分别为横纵坐标。在某个特定电压和频率下，如果诊断测试程序通过了，则标记为绿色（pass），如果未通过，则标记为红色（fail）。

    b. ZeBu: Synopsys 公司推出的高容量仿真系统，支持多种软硬件结合的调试方式，可以处理多数在Verification下具有挑战性的Bug调试。
